# 백룡 기상판단 지원체계

백룡 기상판단 지원체계는 해상기상, 육상기상, 항공기상을 분리해서 운용하는 지도·카드 기반 판단지원 대시보드입니다. 실시간 해양·기상·환경 데이터를 기준으로 권역별 위험도를 산정하고, 각 항목의 기여도와 적용된 판단 규칙을 함께 보여주는 설명가능 인공지능 기반 판단지원체계입니다.

현재 프로젝트는 서버 없이 정적 호스팅 환경에서 실행되도록 구성되어 있으며, API 키나 민감정보를 프론트엔드 코드에 직접 넣지 않는 것을 기본 원칙으로 둡니다.

## 체계 구성

- 백룡 해상기상 판단체계: 해상 권역, 항구, 해수욕장, 관측부이, 원해 지점을 기준으로 해양상황과 해상 접근 취약도를 표시합니다.
- 백룡 육상기상 판단체계: 전국 시군구 단위의 기상, 대기질, 산불, 특보 정보를 기준으로 육상 활동환경 위험도를 표시합니다.
- 백룡 항공기상 판단체계: 항공 관측권역별 풍속, 돌풍, 시정, 운고, 낙뢰 등 항공기상 위험요소를 표시합니다.

## 주요 기능

- 로그인 후 해상기상, 육상기상, 항공기상 화면 선택
- 사이드 메뉴 아코디언을 통한 화면 전환, 공식 기상지도 열기, 운용방법 확인
- 설정 화면에서 해상·육상·항공 측정지역 표시 여부 관리
- 설정 화면의 완전 초기화로 모든 지역 선택 해제 및 기본 상태 복구
- 윈디 지도 기반 상황판과 지도 레이어 선택
- 권역별 위험도 점수, 단계, 색상 카드 표시
- 항목별 기여도 막대그래프와 적용된 판단 규칙 표시
- 해상·육상·항공별 카드, 시간별 기상, 요약 표 구성
- 정적 사이트 배포 가능

## 계산 방법

### 해상기상 판단

해상기상 판단은 시정, 파고, 풍속, 시간대, 조석, 조류, 날씨, 온도, 미세먼지, 운무, 감시장비 시정을 각각 0~1 사이 위험 점수로 변환한 뒤 가중합으로 계산합니다.

- 시정 20, 파고 15, 풍속 15, 시간대 10, 조석 10, 조류 10
- 날씨 5, 온도 3, 미세먼지 4, 운무 5, 감시장비 시정 10
- 낮은 시정, 낮은 파고, 낮은 풍속, 야간 시간대, 만조 접근, 접근 유리 조류는 해상 접근 취약도를 높이는 방향으로 반영합니다.
- 파고나 풍속이 매우 높은 경우에는 위험도를 올리는 대신 해상 이동 제한 요소로 별도 설명합니다.

### 육상기상 판단

육상기상 판단은 4개 축을 먼저 계산한 뒤 최종 점수를 산정합니다.

- 인원 부담도: 체감온도 또는 습구흑구온도, 기온, 습도, 미세먼지, 자외선지수
- 시야 제한도: 시정, 안개·운무, 강수량, 미세먼지, 날씨 상태
- 이동 제한도: 강수량, 강수확률, 풍속, 적설, 결빙 가능성, 시정
- 환경 위험도: 산불지수, 기상특보, 풍속, 낙뢰위험, 기압 추세

최종 육상 활동환경 위험도는 `인원 부담도 × 0.35 + 시야 제한도 × 0.25 + 이동 제한도 × 0.25 + 환경 위험도 × 0.15`로 계산하고, 0~100점 범위로 제한합니다.

### 항공기상 판단

항공기상 판단은 평균풍속, 순간풍속, 풍향, 강수·강설, 시정, 안개·운무, 기온, 습도·이슬점, 낙뢰·뇌우, 구름고도·운량을 0~1 위험 점수로 변환한 뒤 가중합으로 계산합니다.

- 순간풍속과 낙뢰·뇌우는 즉시 위험기상으로 강하게 반영합니다.
- 낮은 시정, 낮은 운고, 높은 운량은 영상·시야 제한과 촬영 여건 악화로 표시합니다.
- 저온·고온과 높은 습도·이슬점은 배터리, 기체 부담, 결로 가능성으로 설명합니다.

모든 체계의 최종 위험도 단계는 40점 미만 `양호`, 40점 이상 70점 미만 `주의`, 70점 이상 `위험`으로 분류합니다.

## 실행

```bash
npm install
npm run dev
```

로컬 확인 주소는 기본적으로 [http://localhost:3000](http://localhost:3000)입니다. 이미 사용 중인 포트가 있으면 `npm run dev -- --port 3001`처럼 다른 포트를 지정합니다.

## 정적 빌드

```bash
npm run build
npm run start
```

`npm run build`는 정적 배포용 결과물을 생성합니다. `npm run start`는 생성된 결과물을 정적 서버로 확인합니다.

## 배포 방법

- 깃허브 페이지
- 넷리파이
- 클라우드플레어 페이지
- 버셀 정적 호스팅

하위 경로에 배포하는 경우 빌드 환경에서 `NEXT_PUBLIC_BASE_PATH`를 저장소 페이지 경로에 맞춰 지정합니다.

## API 및 기준 데이터

현재 화면은 정적 배포를 우선으로 하므로 공개 가능한 기준값과 시연용 권역 데이터를 사용합니다. 해상, 육상, 항공 상황판은 윈디 지도를 기본으로 표시하며, 설정된 측정지역 카드 선택에 따라 지도가 해당 위치로 이동합니다. 기상청 날씨누리 지도와 해양기상기후정보포털은 사이드 메뉴의 각 체계별 아코디언에서 열 수 있습니다.

해상 측정지점은 인천·경기 서해, 충남 서해, 전북 서해, 전남 서남해, 남해·부산·경남, 동해안, 제주 해역, 중국 황해, 중국 동중국해, 원해 관측으로 묶어 관리합니다. 중국 해역 시험 지점은 중국 API를 호출하지 않고, 윈디 또는 허용된 한국 기상자료 연동 범위 안에서만 표시하도록 구성합니다.

기상청 API허브 연동 후보는 다음과 같습니다.

- 지상관측: 기온, 강수, 기압, 습도, 풍향, 풍속, 돌풍, 적설, 구름, 시정, 현상번호
- 해양관측: 해양기상부이·파고부이 기온, 바람, 기압, 습도, 파고, 파주기, 파향, 수온
- 예특보: 초단기·단기예보, 하늘상태, 강수확률, 파고, 기상특보
- 항공기상: 항공관측 풍향, 풍속, 돌풍, 시정, 활주로시정, 현재일기, 운량, 운고, 기온, 이슬점, 습도, 기압
- 생활기상지수: 체감온도, 자외선지수, 대기정체지수
- 한국환경공단 에어코리아: 미세먼지, 초미세먼지, 오존
- 산림청 산불위험예보정보: 산불위험지수, 산불위험등급

공공 API 키가 필요한 경우 GitHub Pages 프론트엔드에 직접 삽입하지 말고, GitHub Actions, 서버리스 함수, 내부망 API 서버 등을 통해 JSON으로 변환한 뒤 정적 사이트가 해당 JSON만 읽도록 구성해야 한다.
